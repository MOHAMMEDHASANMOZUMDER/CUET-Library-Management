Introducing LibraryForge: System Overview

What is LibraryForge?

A **full-stack, modern web application** built to automate and streamline core library operations, transitioning from manual effort to **data-driven efficiency**.

Core Architecture (Three-Tier)

  • Presentation: HTML, CSS, Vanilla JavaScript (Fast, lightweight UI)
  • Application: **Spring Boot 3.x** REST API (Business Logic)
  • Data: **MySQL** Database (JPA/Hibernate)

Motivation: The Crisis of the Traditional Library

The Pre-Exam Chaos

Immense overcrowding, leading to **long, stressful queues** and slow processing. Students often miss out on essential books due to rush and lack of availability tracking.

The Manual Fine Hassle

Fines must be paid manually by visiting a **bank or counter**. This creates a massive **time sink** for students and adds unnecessary administrative load for staff.

Digital Disconnect

No essential online services: **no online renewal**, **no pre-booking** for physical books, and no platform for sharing user-generated study **notes**.

Innovation 1: Prebooking & Fine Payment Systems

The Smart Prebook Workflow

  • **Problem Solved:** Couldn't reserve unavailable books.
  • **Solution:** Queue-based prebooking with status tracking (PENDING, APPROVED, EXPIRED).
  • **Key Innovation:** The **Admin Pickup Modal** requires staff to select the specific physical `Copy_ID` when fulfilling the prebook, ensuring **100% inventory accuracy**.
Image Placeholder: Admin Prebook Pickup Modal UI

Digital Fine Payment (TrxID)

  • **Problem Solved:** Manual, inconvenient payment process.
  • **Solution:** Students submit payment proof via mobile money (bKash/Nagad TrxID).
  • **Key Innovation:** **Two-Step Verification:** Student submits proof $\rightarrow$ Admin approves payment. This ensures security and prevents duplicate submissions using a unique TrxID validation check.

Innovation 2: Collaboration and Automation

Community Note Sharing Platform

  • **Problem Solved:** No system to share study resources.
  • **Functionality:** Students can upload notes (PDF, DOCX) tied to a book's ISBN.
  • **Key Innovation:** **Moderation Workflow:** All notes enter a **PENDING** state and require an **ADMIN approval** before becoming publicly available. File data is securely stored as `LONGBLOB` in the `notes` table.
Image Placeholder: Note Browsing/Download UI

Automation & Notifications

  • Auto Fine Calculation: When a book is returned late, the `BorrowService` automatically calculates the overdue days and generates a fine record (`UNPAID` status), adhering to the **1 taka/day** policy.
  • Real-time Notifications: Centralized system informs students instantly about **Prebook status changes**, **overdue book reminders**, and **fine payment updates**.

Implementation Deep Dive: Backend & Security

Security Architecture (JWT & RBAC)

  • **Authentication:** **JWT** generated by `JwtTokenProvider` on successful login, stored client-side.
  • **Password Security:** Passwords are hashed using **BCrypt** with salt.
  • **Authorization:** **RBAC** enforced via Spring Security's `SecurityFilterChain` and method-level `@PreAuthorize` annotations (ADMIN/STUDENT).
  • **SQL Injection:** Prevented by using **JPA/Hibernate** parameterized queries.

Database Schema Highlights

  • **Fines Table:** Critical fields include `Amount`, `status (UNPAID/PENDING/PAID)`, and **`trXID` (UNIQUE constraint)**.
  • **Borrow Records:** Tracks `Due_Date` (auto-calculated) and `extended_count` (overdue days).
  • **Notes Table:** Includes `fileData` (LONGBLOB) and `status` for moderation.

Implementation Deep Dive: Frontend & Maintainability

Frontend Principles (Vanilla JS)

  • **Lightweight:** Zero reliance on heavy frameworks, resulting in extremely fast load times and minimal bundle size.
  • **Structure:** Modularized JavaScript (`api-service.js`, `auth.js`, `dashboard.js`) for clear separation of concerns.
  • **State Management:** Minimal state managed using **localStorage** (for JWT/User Role) and transient JS variables.
  • **UX:** Implemented **Pagination** and **Debounced Search** for efficient browsing of large book catalogs.

Deployment Readiness

Docker Configuration

Uses **docker-compose** to define the `backend` (Spring Boot) and `mysql` services, ensuring a reproducible, one-command deployment environment.
